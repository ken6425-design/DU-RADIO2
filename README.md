# 📻 DU電台 — 辦公室點歌系統

一台電腦當**放送端**接喇叭放 YouTube 音樂，辦公室同事用各自的手機 / 電腦打開同一個網址當**點歌端**，
貼 YouTube 連結或搜歌名就能加入排隊播放。內建留言牆、大螢幕飄字、已放送紀錄、點歌排行、收聽人數。

> 適用：大家在**同一個辦公室網路（同一 WiFi / 區域網路）**。放送端那台電腦同時也是小伺服器。

---

## 一、安裝（只需做一次，在放歌那台電腦上）

1. 安裝 [Node.js](https://nodejs.org)（建議 18 以上，裝 LTS 版即可）。
2. 把這個 `office-jukebox` 資料夾放到電腦上，打開「終端機 / 命令提示字元」，進到這個資料夾。
3. 安裝相依套件：

   ```
   npm install
   ```

## 二、啟動

```
npm start
```

啟動後終端機會印出幾個網址，例如：

```
點歌端（同事打開這個）：
  本機： http://localhost:3000
  區域網路： http://192.168.1.50:3000      ← 把這個發給同事

放送端（接喇叭那台電腦打開）：
  http://localhost:3000/player.html

管理 / 放送端密碼： 0000
```

## 三、怎麼用

- **放歌的那台電腦**：瀏覽器打開 `http://localhost:3000/player.html`，輸入密碼（預設 `0000`），
  接上喇叭。之後清單裡有歌就會自動播放，一首放完自動接下一首。
- **同事點歌**：手機或電腦打開區域網路網址（如 `http://192.168.1.50:3000`），可以：
  - **貼連結**：把 YouTube 網址貼上去 → 送進候播清單
  - **搜歌名**：直接搜「歌名 / 歌手」→ 點搜尋結果即可排隊
  - 也能在大螢幕**飄字留言**、用**留言牆**、看**點歌排行**
  - 點右上角「分享」會顯示**掃碼 QR**，同事掃一下就進來了

---

## 四、一個網址、自動分流

打開網址會先問「**我要點歌 / 我是放送端**」：
- 放歌那台選「放送端」→ 按開台（**免密碼**）。
- 同事選「點歌」→ 貼連結或搜歌名排隊，**人人都能拖曳排序、插播、移除**，即時同步。

**沒人點歌時會自動接播**：從內建約 200 首（中／韓／英平均）隨機挑、即時搜尋當下有效影片播放，
不會卡在失效連結、也較少重複。一有人點真歌就立刻接手。


## 五、常見設定（選填）

啟動時可用環境變數調整：

| 變數 | 作用 | 預設 |
|------|------|------|
| `PORT` | 服務埠號 | `3000` |
| `ADMIN_PASS` | 放送端 / 管理（跳過、移除歌曲）密碼 | `0000` |
| `PUBLIC_DIR` | 介面風格資料夾 | `public`（冷色調） |
| `YT_API_KEY` | 填了搜尋更穩定（用官方 YouTube Data API） | 空（用網頁解析，免 key） |

範例（macOS / Linux）：
```
ADMIN_PASS=8888 PORT=4000 PUBLIC_DIR=public-mono npm start
```

### 關於「搜歌名」
- 預設不需要任何 API key，靠解析 YouTube 搜尋頁運作。萬一哪天 YouTube 改版導致搜不到，
  改用「貼連結」永遠有效；或申請一個免費的 **YouTube Data API v3** key 填到 `YT_API_KEY` 更穩。

---

## 六、小提醒
- 清單、紀錄存在記憶體裡，**重啟伺服器會清空**（辦公室場景剛好每天清爽開始）。
- 想讓外網（不同辦公室 / 在家）也能點歌，要另外做內網穿透或部署到雲端，預設只在同一網路內可用。
- YouTube 影片若設了「禁止嵌入播放」，少數歌可能無法在放送端播出，換個版本的連結即可。
