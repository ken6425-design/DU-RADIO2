# 🚀 DU電台 — 雲端部署步驟（免費、一勞永逸）

部署完你會得到一個**固定網址**（例如 `https://du-radio.onrender.com`）。
之後不管哪台電腦、開機還原、VPN、不同 IP —— **大家點開這個網址就能用**，
放歌那台也只要打開網址就行，不用裝任何東西。

整個過程都在瀏覽器點一點，**不需要任何指令**。分兩段：先把程式放上 GitHub，再用 Render 部署。

---

## 第一段：把程式放上 GitHub（當作雲端的程式倉庫）

1. 到 https://github.com 註冊一個免費帳號（有的話直接登入）。
2. 右上角「＋」→「**New repository**」。
   - Repository name 隨便取，例如 `du-radio`
   - 選「**Public**」
   - 按「**Create repository**」
3. 進到新倉庫頁面，點中間的「**uploading an existing file**」連結
   （或 Add file → Upload files）。
4. 把 `DU電台雲端版` 資料夾裡的**所有東西**拖進去上傳：
   - `server.js`、`package.json`、`render.yaml`、`README.md`
   - 以及整個 `public` 資料夾（裡面有 index/point/player.html 和 style.css）
   - （不用上傳 `node_modules`，本來就沒有也沒關係）
5. 下方按「**Commit changes**」。上傳完成。

> 小技巧：在電腦上先進到 `DU電台` 資料夾，全選裡面的檔案和資料夾，一起拖進 GitHub 的上傳框即可。

---

## 第二段：用 Render 部署（免費）

1. 到 https://render.com 用剛剛的 **GitHub 帳號登入**（Sign in with GitHub，最快）。
2. 右上「**New +**」→「**Blueprint**」。
3. 選剛剛建立的 `du-radio` 倉庫 →「**Connect**」。
   - Render 會自動讀到 `render.yaml`，幫你設定好一切。
4. 按「**Apply**」/「**Create**」開始部署，等 1～3 分鐘（第一次比較久）。
5. 完成後，在服務頁面上方會看到網址，長得像：
   ```
   https://du-radio.onrender.com
   ```
   這就是你的固定網址了！

> 如果沒有自動讀到 render.yaml，改用 New + → **Web Service** → 選倉庫 →
> Build Command 填 `npm install`、Start Command 填 `npm start` → Create 即可。

---

## 怎麼用

- **大家只要記一個網址**：`https://du-radio.onrender.com`
  打開後會出現兩顆按鈕：
  - 放歌那台（接喇叭）→ 點「**我是放送端**」→ 按開台（免密碼）。
  - 同事 → 點「**我要點歌**」→ 貼連結或搜歌名排隊，還能拖曳排序、插播。
  **不限網路、不限 IP，VPN 也行。**

---

## 要知道的兩件事

1. **免費方案會休眠**：超過約 15 分鐘沒人用，服務會睡著；下次有人打開時要等
   約 30～50 秒喚醒（畫面會轉一下），醒來後就順了。介意的話可升級 Render 付費方案常駐。
2. **資料不長存**：候播清單、紀錄存在記憶體，服務重啟（或休眠喚醒）會清空，
   每天等於重新開始，辦公室用剛好。

---

## 想改設定？
在 Render 服務的「**Environment**」分頁加環境變數：
- `YT_API_KEY` = 你的 YouTube Data API 金鑰（選填，讓搜尋更穩）

改完按 Save，服務會自動重新部署。
